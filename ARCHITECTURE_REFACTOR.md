# 架构重构 - 轻量化设计

## 问题识别

**原设计的问题**：
- 要求16GB RAM + 8核CPU
- 依赖Docker、Nomad、Consul等重量级组件
- 无法在小型设备（树莓派、VPS等）运行
- **违背了"本地根节点"的轻量化初衷**

**当前设备环境**：
- CPU: 1核
- 内存: 921MB
- 磁盘: 3.4GB剩余
- 无Docker

---

## 重构目标

**核心理念**：
- 本地设备运行"根节点"（协调器）
- 根节点极简、低资源占用
- 重任务全部云端执行
- 支持小型设备（512MB RAM起）

**新的架构原则**：
1. 极简 > 功能丰富
2. 云端执行 > 本地执行
3. 轻量级 > 完整功能
4. 实用 > 完美

---

## 重构方案

### 本地根节点
- Python + Flask（轻量API）
- SQLite（状态存储）
- 简单任务队列
- 内存占用 < 100MB

### 云端执行
- GitHub Actions（主要后端）
- Cloud Run（可选）
- 无需本地Docker
